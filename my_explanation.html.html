HTML is a markup language, and here I was repeating ,because this language it's often confused with a programming language, but it's not. Now, you don't understand why this it's not a programming language. Let me explain. HTML along with XML (eXtensible Markup Language) or XHTML (eXtensible Hyper Text Markup Language) and SGML (Standard Generalized Markup Language) do not describe algorithms, but information, the commands being executed by the browser not by the processor (The language programming is executed by processor). HTML is used to create web pages that can be displayed in a browser. The purpose of HTML is to present information -paragraphs, fonts, tables and so on- rather than to describe the semantics of the document. HTML contains data, organized just like a pdf, word, or a image, which also contains data. If HTML it's a programming language, then this documents are also programming languages, which is totaly wrong. Even tags, for exemple, <p> are data, more exactly, metadata. Yes, <h1> - <h2>, <p> are metadata. I give you a few definition to understand.

What is HTML?
HTML is a form of markup aimed at presenting text documents on a single page using specialized rendering software called an HTML user agent, the best example of such software being the web browser.
What is a programming language?
A programming language is a valid set of expressions and rules (or techniques) for formulating instructions for a "computer". A programming language has a defined set of semantic and syntactic rules. It allows the programmer to specify exactly and announced actions that the computer must perform, in order with data. The specification basically consists in drawing up\writing the necesary programs (programming).
What is metadata?
Metadata are data who describe other data, such as paragraphs. A colection of data. ( <p> ) that are data that presents other data.
